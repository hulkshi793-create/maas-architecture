Wasm 插件热更新全流程

改 Go 代码 → 编译 → 推送镜像 → 改 CRD → xDS 下发 → Envoy 热加载 · 全程零中断

各阶段说明

阶段一：开发者本地

  • ① 修改插件 Go 源代码（auth/main.go）
  • ② 用 TinyGo 编译为 .wasm 二进制（target wasip1）
  • ③ 打包为 OCI 镜像（FROM scratch + COPY auth.wasm）并推送到镜像仓库
  • ④ 新 tag（v2）标识版本，回滚即切回旧 tag

阶段二：K8s 下发

  • ① kubectl apply WasmPlugin CRD（指定 url 为新 v2 镜像和插件配置）
  • ② K8s API Server 校验格式并写入 etcd
  • ③ higress-controller Informer/Watch 机制检测到 CRD 变更
  • ④ Controller 将 WasmPlugin CRD 转换为 Istio 配置对象

阶段三：控制面

  • ① Pilot 接收转换后的 Istio 配置，对比版本号
  • ② 检测到 auth 插件版本号从 v1 变为 v2
  • ③ 仅生成 auth 插件的 xDS Delta 增量（不改其他 99 个插件）
  • ④ 通过 gRPC 长连接向 Envoy 推送 Delta 变更

阶段四：Envoy 热加载

  • ① Envoy 通过 gRPC 长连接收到 xDS Delta，识别 auth 插件需要更新
  • ② 异步从 OCI 镜像仓库拉取新的 auth:v2.wasm
  • ③ 创建新的 Wasm 实例 B，调用 proxy_on_configure() 初始化
  • ④ 将实例 B 注入 HTTP Filter Chain，新请求全部路由到 B
  • ⑤ 旧实例 A 继续处理尚未完成的 in-flight 请求（引用计数保护）
  • ⑥ 旧实例 A 引用计数归零后销毁，热更新完成

Wasm 插件热更新 · kubectl apply → xDS Delta → Envoy 零中断替换 · 全程无 restart
