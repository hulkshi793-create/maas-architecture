Envoy 核心组件架构

监听器 → 过滤器链 (HTTP) → 集群 → 端点 · 参考: jimmysong.io/envoy-handbook

组件说明

Listener（监听器）

监听指定 IP:Port，接受下游客户端连接。支持 TCP、HTTP、gRPC 等协议，负责 TLS 终止。一个 Envoy 可配置多个 Listener。

Filter Chain（过滤器链）

请求处理核心。MaaS 只用 HTTP Filter Chain，17 个 Wasm 插件串行执行。Envoy 原生还支持 Network Filter（TCP 代理等）和 Listener Filter，但 MaaS 作为 API 网关不需要。Wasm 插件在 Envoy 内置沙箱中运行，内存/故障隔离。

Cluster（集群）

上游服务集合。通过 CDS/EDS 动态获取配置，提供负载均衡（轮询/最少连接）、主动+被动健康检查、熔断防级联故障。每个 Worker 线程独立连接池，无锁设计。

Endpoint（端点）

集群中的具体服务实例（IP:Port），最小调度单元。通过 EDS 协议从控制面动态推送，支持扩缩容和滚动更新，毫秒级感知变化。
