Envoy 核心组件架构

Listener → Network Filter → HTTP Filter Chain → Cluster → Endpoint (上游)

组件说明

Listener（监听器）

监听 IP:Port，接受下游连接，完成 TLS 终止。TLS 证书由 SDS 动态推送。解密后流量交给 Network Filter。

Network Filter + HCM

HTTP 连接管理器 (HCM) 是核心网络过滤器，负责 HTTP 协议解析、HTTP/2 多路复用、gRPC 支持。HCM 内部驱动 HTTP Filter Chain 执行。Envoy C++ 原生实现。

HTTP Filter Chain

MaaS 过滤器链，双向执行——请求方向依次串行，响应方向反向再过一遍。auth/rate-limit 等为 Wasm 插件 (沙箱隔离，支持热更新)，routing-forwarding 为 Envoy 原生 C++ 过滤器，路由规则由 RDS 下发。

Cluster + Endpoint

Cluster Manager 在 Envoy 进程内，管理 LB、健康检查（后台探测）、连接池（每 Worker 独立无锁）、熔断 (Envoy 原生)。Endpoint 在进程外，为上游实例 IP:Port，通过 EDS 动态同步。
