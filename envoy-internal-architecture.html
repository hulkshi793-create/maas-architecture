Envoy 核心组件架构

Listener → Network Filter → HTTP Filter Chain → Cluster → Endpoint（上游）

各层说明

Listener · 监听器

监听 IP:Port，接受下游连接，完成 TLS 终止。证书由 SDS 动态推送，解密后流量交给 Network Filter。

Network Filter · 网络过滤器

HTTP 连接管理器 (HCM) 负责协议解析、HTTP/2 多路复用。HCM 为 Envoy C++ 原生实现，内部驱动 HTTP Filter Chain 执行。

HTTP Filter Chain · 双向串行

请求方向依次串行执行各过滤器，响应返回时反向再执行一遍。auth/rate-limit 等为 Wasm 扩展（沙箱隔离），routing-forwarding 为 C++ 原生。路由规则由 RDS 下发。

Cluster · 端点

Cluster Manager（进程内）管理 LB、连接池（每 Worker 独立无锁）、原生熔断、后台健康探测。Endpoint（进程外）为上游实例 IP:Port，通过 EDS 动态同步。
