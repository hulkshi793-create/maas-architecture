Envoy 核心组件架构

监听器 → 网络过滤器 → HTTP 过滤器链（双向） → 集群 → 端点 · 参考 jimmysong.io

组件说明

Listener（监听器）

监听指定 IP:Port，接受下游连接。通过 LDS 动态获取配置，通过 SDS 动态获取 TLS 证书。TLS 终止在 Listener 层完成，解密后的明文流量交给 Network Filter。

Network Filter（网络过滤器）

处理 TCP/UDP 原始流量。核心组件是 HTTP 连接管理器 (HCM)，负责 HTTP 协议解析、多路复用。HCM 内部再启动 HTTP Filter Chain。Envoy C++ 原生实现。

HTTP Filter Chain（双向）

MaaS 17 个插件在此链上串行执行。请求方向依次经过，响应方向反向再执行一遍（统计、改响应头等）。auth/rate-limit 等为 Wasm 插件（沙箱隔离），routing-forwarding 为 Envoy 原生 C++ 过滤器。路由规则由 RDS 动态下发。

Cluster Manager + Endpoint

Cluster Manager（Envoy 进程内）管理负载均衡、健康检查、连接池、原生熔断。每个 Worker 线程独立连接池，无锁设计。健康检查为独立后台探测，不经过业务请求链路。Endpoint 为具体上游实例，通过 EDS 动态推送，在 Envoy 进程外。
