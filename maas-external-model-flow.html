MaaS 外接厂商模型 — 请求→回复全链路架构

Higress Gateway · 17 插件链 · 内部 vLLM / 外部厂商双路径 · ALS 日志管线

内部模型路径

  • • Client → Higress → vLLM 直通
  • • 无需协议转换（OpenAI 兼容）
  • • auth 插件校验内部 Key
  • • Redis 计数 → ALS → Kafka → ES
  • • ~5000 Agent, 400 服务 × 8 GPU

外部厂商路径

  • • Client → Higress → Adapter → 厂商
  • • Adapter 做协议转换 + 厂商 Key 注入
  • • 双鉴权：网关 Key + 厂商 Key
  • • 需接内容安全（阿里绿网等）
  • • 计费需网关计数 + 厂商账单对账

当前缺口 (接外部必需)

  • • 动态 API Key 管理 (auth 静态)
  • • 请求体改写 / 协议转换服务
  • • 内容安全审核 (P1)
  • • 响应语义缓存 (P1)
  • • 模型级健康检查 + 自动摘除
