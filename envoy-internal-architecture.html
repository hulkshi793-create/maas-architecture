Envoy 核心组件架构

监听器 → 过滤器链 (网络层→应用层) → 路由 → 集群 → 端点 · 参考: jimmysong.io/envoy-handbook

组件说明

Listener（监听器）

监听指定 IP:Port，接受来自下游客户端的连接请求。支持 TCP、HTTP、gRPC 等多种协议。一个 Envoy 可配置多个 Listener，每个绑定独立的 Filter Chain。

Filter Chain（过滤器链）

请求处理的核心。链式执行一系列过滤器，处理和转换请求/响应。包含三类过滤器：HTTP Filter（路由、认证、限流等 L7 处理）、网络过滤器（TCP 代理、TLS 等 L3/L4 处理）、Other Filters（Dubbo、Thrift 等协议）。Wasm 插件是 HTTP Filter 的一种实现方式，在 Envoy 内置的 Wasm 沙箱中执行。

Cluster（集群）

定义上游服务的集合。提供负载均衡（轮询/最小请求/最少连接）、主动和被动健康检查、熔断机制防止级联故障、连接池管理。每个 Worker 线程维护独立连接池，无锁设计。

Endpoint（端点）

集群中的具体服务实例（IP:Port），集群的最小调度单元。通过 EDS 协议从控制面动态推送变更，支持实例扩缩容和滚动更新，毫秒级感知端点变化。
